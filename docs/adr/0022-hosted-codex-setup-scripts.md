# Hosted Codex Setup Scripts

EggDoc maintains thin Shell and PowerShell setup scripts for Codex + EggAi because Codex integration should be a repeatable install flow, not only a manual configuration tutorial. Shell installs and verifies Codex CLI on macOS, Linux, and WSL. PowerShell installs the official OpenAI Codex desktop app from Microsoft Store product `9PLM9XGG6VKS` and verifies package family `OpenAI.Codex_2p2nqsd0c76g0`. EggAi mode validates inputs, probes `/models`, verifies a selected model with a minimal Responses request, and updates `config.toml` with `env_key = "EGGAI_API_KEY"`. It does not run `codex login`, write `auth.json`, or replace an existing OpenAI/ChatGPT login.

The Shell wrapper requires Node.js 24.18.0 or newer, installs Node 24 LTS from `nodejs.org` when the runtime is missing or too old, and installs `@openai/codex@latest` with a user-level npm prefix. The Windows wrapper uses the exact Microsoft Store product above and does not install Node.js. Official Node.js and Store failures terminate installation without an alternate binary source. The CLI npm command uses `https://registry.npmmirror.com` by default without changing persistent npm configuration. EggAi API reachability remains independent from official account login reachability.

# Hosted Codex Setup Scripts

EggDoc maintains thin Shell and PowerShell setup scripts for Codex + EggAi because Codex integration should be a repeatable install/update flow, not only a manual configuration tutorial. The scripts download and verify the official Codex installer, validate EggAi inputs, probe the selected EggAi `/models` endpoint before changing local files, and update `config.toml` with a provider-scoped `env_key = "EGGAI_API_KEY"`. They do not run `codex login`, write `auth.json`, or replace an existing OpenAI/ChatGPT login. An authenticated tutorial panel generates commands from the selected API Credential and exact model ID, with `zh-cn` as the default language.

The official installer URL is configurable through `CODEX_INSTALLER_URL` for a locally reachable mirror or enterprise proxy. This separates binary installation reachability from API reachability: EggAi mode requires the selected EggAi endpoint to respond, but never requires an official OpenAI account or official API login.

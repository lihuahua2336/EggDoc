# Hosted Claude Code setup scripts install the official npm package

EggDoc hosts Shell and PowerShell Claude Code Setup Scripts that validate an npm release target, require Node.js 22 or newer, install Node 22 automatically when needed, install `@anthropic-ai/claude-code`, persist the npm global command directory on PATH, and verify `claude --version`. Shell downloads Node only from `nodejs.org`; Windows uses exact winget package `OpenJS.NodeJS.LTS`. The npm command uses `https://registry.npmmirror.com` by default without changing persistent npm configuration. Default mode does not modify Claude Code settings.

When a Reader explicitly selects `EggAi 配置`, the same scripts first send a bounded streaming tool-use request to the normalized `/v1/messages` endpoint with the Selected API Credential and the selected main model. Installation and configuration continue only after the endpoint returns Anthropic SSE events, a Messages ID, a tool-use block, and a message-stop event. The scripts then back up and structurally merge the Reader's user-level `settings.json`, write the credential as `ANTHROPIC_AUTH_TOKEN`, and write distinct main, Sonnet, Opus, Haiku, and Fable model mappings. Generated commands only select EggAi's confirmed model identifiers: `claude-sonnet-5`, `claude-sonnet-4-6`, `claude-opus-4-8`, `claude-opus-4-7`, `claude-opus-4-6`, `claude-fable-5`, and `claude-haiku-4-5`. The main model priority is Fable 5, Sonnet 5, the highest available Opus, Sonnet 4.6, then Haiku 4.5. Family-specific role mappings still prefer their own highest supported model before falling back to the selected main model. The scripts also remove a conflicting managed `ANTHROPIC_API_KEY` and preserve unrelated settings.

This runtime check verifies the Reader's real URL, Bearer credential, main model, streaming transport, and tool-use path. A missing or outdated Node.js runtime is an installation prerequisite and may be installed before the check; the Claude npm package and settings are not changed until the check succeeds. The persisted `ANTHROPIC_AUTH_TOKEN` takes precedence over a saved claude.ai login, so Claude Code uses EggAi without a second login while the configuration remains present. The check does not automate Claude Code's interactive `/status` display or every role-specific long-running workflow. EggDoc accepts that residual release risk because the requested flow must remain one command and fail before Claude package installation or settings changes when the critical gateway contract is absent. The generated command must warn about the verification request and that the credential is present in command history and plaintext settings. This EggAi mode supersedes the earlier decision that the setup scripts never modify Claude configuration.

# Hosted Claude Code setup scripts delegate installation to Anthropic

EggDoc will host Shell and PowerShell Claude Code Setup Scripts that validate an official release target, reject empty or HTML installer responses, invoke Anthropic's recommended native installers, and verify `claude --version`. The default mode delegates binaries, updates, authentication, and configuration to Anthropic and does not modify user settings.

When a Reader explicitly selects `EggAi 配置`, the same scripts first send a bounded streaming tool-use request to the normalized `/v1/messages` endpoint with the Selected API Credential and the selected main model. Installation and configuration continue only after the endpoint returns Anthropic SSE events, a Messages ID, a tool-use block, and a message-stop event. The scripts then back up and structurally merge the Reader's user-level `settings.json`, write the credential as `ANTHROPIC_AUTH_TOKEN`, and write distinct main, Sonnet, Opus, Haiku, and Fable model mappings. The generated command prefers `claude-sonnet-5` for main and Sonnet, `claude-opus-4-8` for Opus, and `claude-fable-5` for Fable and the otherwise unavailable Haiku role; it falls back to the highest available model in each family and finally to the main model. The scripts also remove a conflicting managed `ANTHROPIC_API_KEY` and preserve unrelated settings.

This runtime check verifies the Reader's real URL, Bearer credential, main model, streaming transport, and tool-use path. The persisted `ANTHROPIC_AUTH_TOKEN` takes precedence over a saved claude.ai login, so Claude Code uses EggAi without a second login while the configuration remains present. The check does not automate Claude Code's interactive `/status` display or every role-specific long-running workflow. EggDoc accepts that residual release risk because the requested flow must remain one command and fail before installation or settings changes when the critical gateway contract is absent. The generated command must warn about the verification request and that the credential is present in command history and plaintext settings. This EggAi mode supersedes the earlier decision that the setup scripts never modify Claude configuration.

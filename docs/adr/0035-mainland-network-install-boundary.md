# Mainland network boundary for hosted installers

EggDoc separates installation transport from provider authentication.

Hosted Shell and PowerShell entry downloads use bounded retries/timeouts and report non-zero failures. Codex CLI and Claude Code CLI use their official native installer URLs; Windows Codex uses exact Microsoft Store product `9PLM9XGG6VKS` and verifies the OpenAI AppX package family. EggDoc does not provide alternate binary packages. Installer subprocesses receive a process-scoped `https://registry.npmmirror.com` npm registry default for mainland npm reachability, without changing persistent npm configuration.

EggAi mode has a different success contract. Codex probes the selected EggAi `/models` endpoint with the selected Bearer key before changing local files. Claude probes the selected EggAi Anthropic Messages endpoint with a bounded streaming tool-use request before installation and settings changes. Successful EggAi configuration therefore depends on the EggAi endpoint being reachable, not on an OpenAI or Anthropic account login.

The scripts do not promise that official account pages or official API endpoints are reachable from mainland China. Default mode intentionally preserves the official provider flow. Generated commands already use the reachable EggDoc wrapper origin `https://doc.eggai.icu`; readers do not configure an installer mirror. A self-hosted deployment may change `PUBLIC_INSTALLER_ORIGIN` only to its own wrapper origin, never to a binary mirror. If an official installer endpoint or Microsoft Store is unavailable, the wrapper reports the failure and stops. It never silently switches to a different binary source. Shell honors standard `HTTPS_PROXY`/`HTTP_PROXY`/`NO_PROXY` variables; Windows PowerShell uses the system/WinHTTP proxy configuration.

# Mainland network boundary for hosted installers

EggDoc separates installation transport from provider authentication.

Hosted Shell and PowerShell entry downloads use bounded retries/timeouts and report non-zero failures. Codex CLI and Claude Code CLI use the official npm package identities `@openai/codex` and `@anthropic-ai/claude-code`; Windows Codex uses exact Microsoft Store product `9PLM9XGG6VKS` and verifies the OpenAI AppX package family. CLI scripts require a compatible Node.js runtime and automatically install Node 22 from `nodejs.org` or exact winget package `OpenJS.NodeJS.LTS` when needed. EggDoc does not provide alternate binary packages. npm receives a command-scoped `https://registry.npmmirror.com` registry default for mainland npm reachability, without changing persistent npm configuration.

EggAi mode has a different success contract. Codex probes the selected EggAi `/models` endpoint with the selected Bearer key before changing local configuration files. Claude may install the Node.js prerequisite first, then probes the selected EggAi Anthropic Messages endpoint with a bounded streaming tool-use request before installing the Claude npm package or changing settings. Successful EggAi configuration therefore depends on the EggAi endpoint being reachable, not on an OpenAI or Anthropic account login.

The scripts do not promise that official account pages or official API endpoints are reachable from mainland China. Default mode intentionally preserves the official provider flow. Generated commands already use the reachable EggDoc wrapper origin `https://doc.eggai.icu`; readers do not configure an installer mirror. A self-hosted deployment may change `PUBLIC_INSTALLER_ORIGIN` only to its own wrapper origin, never to a binary mirror. If `nodejs.org`, winget, Microsoft Store, or the npm registry is unavailable, the wrapper reports the failure and stops. It never silently switches to a different binary source. Shell honors standard `HTTPS_PROXY`/`HTTP_PROXY`/`NO_PROXY` variables; Windows PowerShell uses the system/WinHTTP proxy configuration.

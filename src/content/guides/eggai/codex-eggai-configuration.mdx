使用安装页中的 `EggAi 配置` 后，脚本会设置 Codex 的模型提供方，并让 Codex CLI 保存所选配置分组的 API Key。

## 配置文件

默认配置目录是 `~/.codex`。Windows 对应 `$HOME\.codex`。如果设置了 `CODEX_HOME`，则使用该目录。

| 文件 | 用途 |
| --- | --- |
| `config.toml` | 保存 EggAi 提供方、Base URL 和默认回复语言 |
| `auth.json` | 由 Codex CLI 保存登录凭据 |
| `config.toml.eggai.bak` | 脚本写入前创建的原配置备份 |

## config.toml

默认生成的 EggAi 配置如下：

```toml
cli_auth_credentials_store = "file"
model_provider = "eggai"
developer_instructions = "请默认使用简体中文回答，除非用户明确要求其他语言。"

[model_providers.eggai]
name = "EggAi"
base_url = "https://api.eggai.icu/v1"
wire_api = "responses"
requires_openai_auth = true
```

- `cli_auth_credentials_store`：让 Codex CLI 使用本地文件保存登录凭据。
- `model_provider`：把当前模型提供方切换为 EggAi。
- `developer_instructions`：设置默认回复语言。
- `base_url`：使用所选 EggAi 配置分组对应的 API 地址。
- `wire_api`：使用 Responses API 协议。
- `requires_openai_auth`：让该提供方读取 Codex 的登录凭据。

## API Key

API Key 不会写进 `config.toml`。安装脚本通过标准输入调用：

```sh
codex login --with-api-key
```

随后由 Codex CLI 将凭据保存到本地认证文件。不要分享 `auth.json`、完整安装命令、终端历史或包含 API Key 的截图。

## 重复配置

每次运行 EggAi 配置前，脚本都会把当前 `config.toml` 备份为 `config.toml.eggai.bak`。然后删除旧的 EggAi 托管配置和旧的 `[model_providers.eggai]`，并替换顶层的 `cli_auth_credentials_store`、`model_provider` 和 `developer_instructions`。除这些项目外，其他原有配置会被保留。

这意味着重复运行会更新 EggAi 配置，不会不断追加重复配置块。

## 检查登录状态

运行以下命令确认 Codex CLI 已读取认证信息：

```sh
codex login status
```

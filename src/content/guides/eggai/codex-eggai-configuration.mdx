在安装页面选择 `EggAi 配置` 后，脚本会先验证所选 API 凭据和模型，再安装 Codex 并更新用户级配置。网页命令中的模型 ID 来自 EggAi `/models` 接口，不会猜测或硬编码一个不存在的模型。

## 配置文件

默认配置目录是 `~/.codex`，Windows 对应 `$HOME\.codex`。设置 `CODEX_HOME` 后，脚本会使用该目录。

| 文件 | 用途 |
| --- | --- |
| `config.toml` | 保存 EggAi 提供方、Base URL、模型和默认回复语言 |
| `eggai.env` | macOS/Linux 保存 `EGGAI_API_KEY` 的受限权限文件 |
| `config.toml.eggai.bak` | 配置发生变化时保留的原配置备份 |

## config.toml

网页生成的配置形状如下，API Key 不会写进 TOML：

```toml
model_provider = "eggai"
developer_instructions = "请默认使用简体中文回答，除非用户明确要求其他语言。"
model = "gpt-5.2"

[model_providers.eggai]
name = "EggAi"
base_url = "https://api.eggai.icu/v1"
env_key = "EGGAI_API_KEY"
env_key_instructions = "EggDoc stores EGGAI_API_KEY in the user environment."
wire_api = "responses"
```

- `model_provider` 选择 EggAi。
- `model` 是 EggAi `/models` 返回的准确模型 ID。
- `env_key` 告诉 Codex 从 `EGGAI_API_KEY` 环境变量读取凭据。
- `wire_api` 使用 Responses API 协议。

脚本不会执行 `codex login`，也不会创建或修改 `auth.json`，因此不会覆盖已有的 OpenAI/ChatGPT 登录状态。

## API Key

macOS/Linux 会将密钥写入 `CODEX_HOME/eggai.env`，并在用户 shell profile 中维护受控加载块。Windows 默认写入当前用户环境变量。PowerShell 命令会显式设置：

```powershell
$env:EGGAI_CODEX_ENV_SCOPE = 'User'
```

`User` 作用域会跨终端保存，`Process` 仅适合测试或临时会话。API Key 仍然会出现在一键命令中，请不要分享命令、终端历史、截图或配置目录。

## 重复配置

每次配置前，脚本会删除旧的 `[model_providers.eggai]` 及其所有子表，然后合并新的托管块，同时保留其他 Codex 设置。只有配置内容发生变化时才创建或更新 `.eggai.bak`；目标配置已是最新状态时不会重写备份。

配置写入使用临时文件和原子替换。写入或环境变量验证失败时，脚本会恢复原配置；Windows 还会恢复原 `EGGAI_API_KEY` 的值及其是否存在状态。

## 中国大陆网络说明

EggAi 模式在修改本地文件前会用所选 API Key 请求 EggAi `Base URL` 的 `/models`。该检查通过后，Codex 运行时直接访问 EggAi，不会执行 `codex login`，也不依赖 OpenAI/ChatGPT 账户网络。

安装器下载是另一条网络链路。官方 Codex 安装地址无法访问时，请在执行前为当前终端设置 `CODEX_INSTALLER_URL`，指向组织内经过审核的可达镜像；Shell 也会透传 `HTTPS_PROXY`、`HTTP_PROXY` 和 `NO_PROXY`。如果是 EggDoc 页面生成的命令，请将部署环境的 `PUBLIC_INSTALLER_ORIGIN` 配置为中国用户可访问的自定义域名，并可通过 `PUBLIC_CODEX_INSTALLER_ORIGIN` 自动按平台注入正确的 `.sh` 或 `.ps1` 镜像地址。

因此，“EggAi 配置成功”表示 EggAi 端点、模型和本地配置验证通过；它不表示官方账户登录或官方账户 API 在当前网络中可用。
